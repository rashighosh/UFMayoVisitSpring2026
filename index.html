💻 VERG Tech Session

🏗️

Working with Agentic AI

Structure: 5 min intro → 20 min build → 5 min share-out. Split into 2 mixed groups (UF + Mayo in each). Each group goes to a station led by a tech person. Everyone participates in building something live. At the end, each group does a 90-second demo. Room votes on the most useful workflow.

The Big Idea Traditional software does what you tell it to. Agentic AI figures out what needs to happen, breaks it into steps, calls the right tools, and hands you a result. Each group demonstrates a different agentic pattern.
🟢 Group 1 · Claude

UI/Feature Mockups — Quickly iteration of interface ideas

Current challenge: We currently spend hours in Figma for rough UI mockups.

Goal: Turn plain text notes / feature requests from meetings into 2–3 quick UI mock‑ups using an AI agent.

Task: Grab 1-2 real feature request notes → Craft a prompt that asks the AI to generate multiple UI mock‑ups (e.g., low‑fidelity layouts, component ideas). → Note: structural constraints (mobile vs web, key sections, tone).

Agentic concept: Using no-code AI tools to quickly turn ideas into multiple visual pathways.

🔧 Claude
🟡 Group 2 · NotebookLM

Quarterly Reports - Quickly generate tech dev updates

Current challenge: Every quarter, tech dev re‑assembles Google Doc + GitHub info manually for report updates.

Goal: Use an agentic AI (e.g., NotebookLM reading our docs + GitHub) to auto‑generate structured quarterly‑style reports.

Task: Pick one sample quarter’s worth of data (commits, issues, Google Doc text) → Craft a prompt to fill out a template for tech dev updates (new features, maintenance, progress, blockers, next steps)

Agentic concept: Using RAG (retrieval augmented generation) to pull from relevant sources and generate relevant information

🔧 NotebookLM
Share-Out + Takeaways Each group shares/presents what they built, hopefully demonstrating how we can use agentic AI to save time + iterate faster + address manual processes!
